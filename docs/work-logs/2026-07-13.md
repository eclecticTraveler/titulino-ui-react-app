### T12 — Logo click → landing page

### T13 — Home icon in header
`src/components/layout-components/HeaderNav.js` — added `<HomeOutlined />` icon linked to `/lrn/landing` on the right side of the header (before the search icon), shown only on desktop when flag is on. "Inicio" text item was NOT added to top nav.

### T14 — Route wiring for `/lrn/landing`
`src/views/app-views/index.js` — added `<Route path="landing" element={<RouteElement component={LandingPage} />} />` and updated the empty-path `<Navigate>` to point to `landing` when flag is on.

### T15 — Replace old split-screen with card panel
`src/views/landing/index.js` — changed import from `LandingUnauthenticated` (old 60/40 split-screen) to `EnrollmentUnauthPanel` (Google/Facebook OAuth cards + course tiles). The new panel is now the unauthenticated landing for both the `/lrn/landing` route and after sign-out.

### T16 — Navigation bug fix (critical)
Root cause: `SharedCourseLevel` and `AuthCourseLevel` had `IS_ENROLLMENT_LANDING_ON` redirect blocks that intercepted any navigation to a course URL when the flag was on.

Removed `IS_ENROLLMENT_LANDING_ON` conditional blocks from:
- `src/views/shared-views/course-level/index.js` — two blocks: unauthenticated case (was showing `EnrollmentUnauthPanel`) and not-enrolled authenticated case (was showing `LandingAuthenticatedHub`)
- `src/views/auth-views/course-level/index.js` — one block: not-enrolled case (was showing `LandingAuthenticatedHub`)

Also removed now-unused imports (`EnrollmentUnauthPanel`, `LandingAuthenticatedHub`, `env`) from both files.

**Rule established:** `IS_ENROLLMENT_LANDING_ON` controls only: the default empty-path redirect, the post-login fallback, the logo link, and the home icon. It must not override rendering at explicit course routes.

### T17 — Delete obsolete file
Deleted `src/views/landing/LandingUnauthenticated.js` — no longer imported anywhere.
