## Missive audiencemessages crash — root cause and fix

### Root cause
`GcpBucketAdapter` constructor called `GetSecretAsync("GCP_AUTH_MASTER_TOKEN").GetAwaiter().GetResult()` — a synchronous block on an async gRPC call to GCP Secret Manager. On `pd-titulino-lang` the OAuth2 access token for the service account intermittently expires and can't be refreshed (Google token endpoint times out). When that happens, the blocking call never returns, the process crashes during DI construction, and `audiencemessages` exits before sending anything.

The two exception patterns in the log:
- `StatusCode="DeadlineExceeded"` — gRPC to Secret Manager timed out
- `The access token has expired and could not be refreshed. Errors: timeout, timeout, timeout` — token refresh failing

### Fix applied
`C:\Users\AlbertoArellano\source\repos\titulino-communication\Repository\Provider\GcpBucketAdapter.cs`

Removed the credential fetch from the constructor. Added `EnsureInitializedAsync()` (double-checked lock, `SemaphoreSlim`) and called it at the top of every method that uses `_storageClient` or `_googleCredential`. Methods that only use `_httpClient` (template fetches, welcome messages, audience messages) bypass Secret Manager entirely.

**Effect on `audiencemessages`**: that subcommand only calls HTTP-only bucket methods. After the fix it never touches GCP Secret Manager — the crash is gone.

Committed: `6934118` on branch `categoryCommuncationRefactor` in `titulino-communication`.
