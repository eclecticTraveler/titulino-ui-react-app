## Domain: Frontend — KnowMeV3 navigate-away bug

### Summary

Fixed three issues in `src/components/layout-components/KnowMeV3.js` that caused students returning to the Know Me page to see a blank form instead of their review status.

### Root causes found

1. **`failed` AI status unhandled** — `ACTIVE_STATUSES` only contained `['pending', 'processing']`; `'failed'` fell through `!isCompleted` → showed empty form with no message.
2. **Form shown during `pending`** — `!isCompleted` was true during `pending`/`processing`, so an empty form appeared below the "AI Review in Progress" banner.
3. **No answer recovery** — answers were not persisted anywhere; if student navigated away during `pending` and came back to a `failed` state, there was nothing to resubmit.

### Fixes applied

| Fix | Change |
|-----|--------|
| `failed` state | Added `isFailed = aiStatus === 'failed'` → red error Alert + "Resubmit" button |
| Hide form during pending | `{!isCompleted && (` → `{!isCompleted && !isPending && (` |
| localStorage answer recovery | On submit: save text answers to `knowme_answers_${theme}_${chapter}_${email}`. On `failed` fetch: restore via `form.setFieldsValue`. On `completed` fetch: clear key. |

### Navigation scenarios — after fix

| Student returns when AI is… | What they see |
|-----------------------------|---------------|
| `pending` / `processing` | "AI Review in Progress" info banner only (no empty form) |
| `completed` | Results view with corrected essays and feedback |
| `failed` | Red error alert + form pre-filled with their original answers + "Resubmit" button |
| `not_found` (never submitted) | Empty form as normal |
