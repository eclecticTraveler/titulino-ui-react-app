## Changes shipped

### T1 — Token gap fix (`src/managers/LrnManager.js` ~line 471)
Added `shouldRetrieveRegistrationToken: true` to the `resolveEnrollmentProfileContext` call inside `getEnrollmentProfilePictureRequirement`.

**Why:** Without this, unauthenticated found-enrollees (matched by email + year, no login) had no token when the GCP pre-check ran. The check silently fell back to `requiresUpload: true`, making smart mode behave identically to force mode for that path. With this flag, `resolveEnrollmentProfileContext` fetches a service JWT so the GCP check can proceed.

### T2 — Flag flip (`src/configs/EnvironmentConfig.js` line 5)
`IS_TO_FORCE_ENROLLMENT_PROFILE_PICTURE_UPLOAD`: `true` → `false`

**Why:** Force mode (always ask for a photo) was appropriate for the initial collection phase. Now that production photos are established, smart mode skips the upload field for enrollees who already have a photo.
