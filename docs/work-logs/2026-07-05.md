## Architecture decisions recorded today

- **Upsert convention:** one `upsert_` function (not separate create + update); use `INSERT ... ON CONFLICT` not procedural IF/ELSE
- **Composite type convention:** inner functions accept/return composite type; wrapper builds `ROW(...)::"Schema".type`; inner call wrapped in `BEGIN/EXCEPTION`; wrapper also returns full composite type (not a scalar field)
- **Column naming:** PascalCase, no underscores; string PKs for small tables
- **Permission grants:** inner fn = postgres+service_role only; admin wrapper = +titulino_administrator+titulino_super_admin
